# 实施路线图

## 总体计划

```
Phase 1: Core (6-8周)          Phase 2: Box (4-6周)           Phase 3: Enterprise (8-10周)
├─ Config Guardian              ├─ N100 镜像制作               ├─ 集中管理后台
├─ OpenClaw Adapter             ├─ 首次启动向导                ├─ SSO 集成
├─ Web UI (配置向导)             ├─ 硬件集成测试                ├─ 审计日志
├─ Desktop 封装                  ├─ 供应链洽谈                  ├─ 集群部署
└─ Safe Mode                    └─ 包装与品牌                  └─ toB 试点
```

## Phase 1: Core（MVP）

**目标**：发布 Claw Desktop 免费版，建立种子用户群。

**时间**：6-8 周

### Week 1-2: Config Guardian

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 项目骨架搭建 | 后端 | 代码仓库、CI/CD |
| 配置快照系统 | 后端 | Git-like 快照存储 |
| 事务性配置框架 | 后端 | `backup → validate → apply → healthcheck → commit` |
| JSON Schema 验证 | 后端 | 配置校验模块 |

**验收标准**：
- 配置修改自动创建快照
- 验证失败不应用配置
- 应用失败自动回滚

### Week 2-3: OpenClaw Adapter

| 任务 | 负责人 | 产出 |
|------|--------|------|
| CRI 接口定义 | 架构 | TypeScript 接口 |
| OpenClaw Adapter 实现 | 后端 | 进程管理、日志获取 |
| 健康检查实现 | 后端 | HTTP 健康检查 |
| 运行时状态管理 | 后端 | 状态机实现 |

**验收标准**：
- 能启动/停止 OpenClaw 进程
- 能获取实时日志
- 健康检查通过

### Week 3-4: Web UI（配置向导）

| 任务 | 负责人 | 产出 |
|------|--------|------|
| UI 框架搭建 | 前端 | React/Vue 项目 |
| 配置表单组件 | 前端 | 动态表单渲染 |
| JSON Schema 表单生成 | 前端 | 根据 Schema 生成表单 |
| API 对接 | 前后端 | 端到端打通 |

**验收标准**：
- 可视化配置替代 JSON 编辑
- 实时校验配置合法性
- 事务性应用配置

### Week 4-5: Tauri 封装

| 任务 | 负责人 | 产出 |
|------|--------|------|
| Tauri 项目搭建 | 前端 | Desktop 基础框架 |
| 后端集成 | 前端 | 内嵌后端服务 |
| 系统托盘 | 前端 | 托盘菜单、启动项 |
| 打包脚本 | 前端 | Windows/macOS/Linux 安装包 |

**验收标准**：
- 双击安装包完成安装
- 系统托盘管理启动/停止
- 开机自启选项

### Week 5-6: Safe Mode

| 任务 | 负责人 | 产出 |
|------|--------|------|
| Safe Mode 服务 | 后端 | 独立进程（端口 18790）|
| 错误诊断模块 | 后端 | 错误码映射、人话解释 |
| Safe Mode UI | 前端 | 恢复操作界面 |
| 自动触发逻辑 | 后端 | Gateway 崩溃检测 |

**验收标准**：
- Gateway 崩溃自动进入 Safe Mode
- 能查看错误原因（人话版）
- 一键回滚功能可用

### Week 6-7: 安全沙盒（Level 1）

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 用户隔离实现 | 后端 | `clawuser` 创建 |
| 权限限制 | 后端 | chroot / landlock |
| 资源限制 | 后端 | cgroup v2 配置 |
| 技能安全扫描 | 后端 | 静态分析、风险提示 |

**验收标准**：
- OpenClaw 运行在隔离用户下
- 无法访问沙盒外文件
- 资源使用受限制

### Week 7-8: 测试与发布

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 集成测试 | QA | 测试用例、报告 |
| 文档编写 | 产品 | 用户手册、安装指南 |
| Alpha 发布 | 全员 | 种子用户招募 |
| 反馈收集 | 产品 | 问题清单、优化项 |

**里程碑**：Claw Desktop v0.1 发布

## Phase 2: Box（硬件版）

**目标**：完成 N100 硬件版量产准备。

**时间**：4-6 周（与 Phase 1 部分并行）

### Week 1-2: N100 镜像制作

| 任务 | 负责人 | 产出 |
|------|--------|------|
| Ubuntu Server 裁剪 | 嵌入式 | 最小化系统镜像 |
| Claw Manager 集成 | 嵌入式 | 预装软件包 |
| 启动优化 | 嵌入式 | 启动时间 < 30s |
| OTA 更新机制 | 嵌入式 | 差分更新 |

**验收标准**：
- 镜像大小 < 2GB
- 启动时间 < 30 秒
- 支持 OTA 更新

### Week 2-3: 首次启动向导

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 设备发现协议 | 后端 | mDNS / 二维码 |
| 初始化 Web UI | 前端 | 手机端适配页面 |
| 扫码配置流程 | 前后端 | 端到端打通 |
| 渠道授权集成 | 后端 | 飞书/微信/钉钉扫码 |

**验收标准**：
- 手机扫码进入配置页面
- 3 步完成初始化配置
- 自动完成 device pairing

### Week 3-4: 硬件集成测试

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 样机测试 | 硬件 | 2-3 家厂商样机对比 |
| 稳定性测试 | QA | 7×24 小时压测 |
| 温度/功耗测试 | 硬件 | 散热方案验证 |
| EMC 测试 | 硬件 | 电磁兼容性 |

**验收标准**：
- 7×24 小时稳定运行
- 表面温度 < 50℃
- 功耗 < 15W

### Week 4-5: 供应链洽谈

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 代工厂评估 | 供应链 | 3 家以上报价 |
| 批量采购协议 | 供应链 | MOQ、交期、价格 |
| 质量协议 | 供应链 | 质检标准、售后 |
| 备件采购 | 供应链 | 电源、外壳、包装 |

**验收标准**：
- 确定主供应商
- 锁定首批 500 台产能
- 单台成本 ¥600 以内

### Week 5-6: 包装与品牌

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 包装设计 | 设计 | 包装盒、说明书 |
| 品牌物料 | 设计 | Logo、贴纸、卡片 |
| 零售页面 | 市场 | 官网产品页 |
| 销售渠道 | 市场 | 电商平台入驻 |

**里程碑**：Claw One 量产准备完成

## Phase 3: Enterprise（企业版）

**目标**：发布 toB 版本，与配套软件打包销售。

**时间**：8-10 周

### 关键功能

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 集中管理 | 设备管理、批量配置 | 中 |
| 安全加固 | Nanobot 默认、WASM 沙箱 | 高 |
| 审计日志 | 操作记录、报表导出 | 中 |
| SSO 集成 | 企业微信/钉钉/飞书 | 中 |
| 集群部署 | 服务器多实例、负载均衡 | 高 |

### 里程碑

- Week 4：内部 Alpha 测试
- Week 6：种子客户试点
- Week 8：toB 配套软件集成
- Week 10：正式发布

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **OpenClaw 大版本不兼容** | 中 | 高 | 版本锁定、社区沟通、预留适配时间 |
| **N100 供应链波动** | 中 | 中 | 备选方案（N95/N200）、多供应商 |
| **开发进度延迟** | 高 | 中 | 敏捷迭代、优先级管理、范围裁剪 |
| **用户反馈不及预期** | 中 | 高 | 早期用户参与、快速迭代、免费版引流 |
| **硬件质量问题** | 低 | 高 | 严格品控、老化测试、保修政策 |
