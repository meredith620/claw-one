# 实施路线图

**最后更新**: 2026-03-20

---

## 总体计划

MVP 阶段（6-8 周）完成 Claw One 核心功能。

```
Phase 1: MVP (6-8周)
├─ Config Guardian (Git 快照)
├─ OpenClaw Adapter (CRI)
├─ Web UI (Vue 3)
├─ systemd 集成
└─ Safe Mode
```

---

## Phase 1: MVP

**目标**：发布 Claw One 可用版本。

**时间**：6-8 周

### Week 1-2: 项目基础

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 项目骨架搭建 | 后端 | Rust 单 crate 项目、CI/CD |
| Axum API 框架 | 后端 | HTTP API 路由、静态文件服务 |
| Vue 3 项目搭建 | 前端 | Vite 项目、Element Plus 集成 |
| 前后端联调 | 全员 | 开发环境打通 |

**验收标准**：
- `cargo run` 启动服务
- `npm run dev` 启动前端
- 浏览器访问 `localhost:8080` 看到页面

### Week 2-3: Config Guardian + Git 快照

| 任务 | 负责人 | 产出 |
|------|--------|------|
| Git 仓库初始化 | 后端 | 自动 `git init`、提交逻辑 |
| 配置保存/读取 | 后端 | JSON 序列化、文件操作 |
| 快照列表 API | 后端 | `git log` 解析 |
| 回滚 API | 后端 | `git checkout` 封装 |

**验收标准**：
- 配置修改自动创建 git commit
- 能列出历史版本
- 能回滚到指定版本

### Week 3-4: Runtime Adapter + systemd

| 任务 | 负责人 | 产出 |
|------|--------|------|
| CRI trait 定义 | 后端 | `ClawRuntime` 接口 |
| OpenClaw Adapter | 后端 | 进程管理、状态查询 |
| systemd 集成 | 后端 | `systemctl start/stop/status` |
| 健康检查 | 后端 | HTTP ping、超时控制 |

**验收标准**：
- 能启动/停止 OpenClaw
- systemd 自动重启生效
- 健康检查 30s 超时可控

### Week 4-5: Web UI

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 配置向导页面 | 前端 | openclaw.json 表单编辑 |
| 状态监控页面 | 前端 | Runtime 状态、日志显示 |
| API 对接 | 前后端 | 端到端打通 |

**验收标准**：
- 可视化配置替代 JSON 编辑
- 实时显示 Runtime 状态

### Week 5-6: Safe Mode

| 任务 | 负责人 | 产出 |
|------|--------|------|
| Safe Mode 状态机 | 后端 | 错误分类、自动回滚逻辑 |
| Safe Mode UI | 前端 | 三场景不同按钮布局 |
| 出厂设置 | 后端 | 重置逻辑、确认弹窗 |
| 日志查看 | 前端 | 错误日志展示 |

**验收标准**：
- 配置错误自动回滚并提示
- 系统错误进入 Safe Mode 让用户决定
- 出厂设置需确认

### Week 6-7: 首次启动向导

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 设备发现 | 后端 | mDNS 或 IP 显示 |
| 初始化向导 | 前端 | 密码设置、模型配置、渠道配置 |
| 首次配置标记 | 后端 | 跳过向导逻辑 |

**验收标准**：
- 首次访问显示向导
- 3 步完成初始化
- 完成后正常进入主界面

### Week 7-8: 测试与优化

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 跨平台测试 | 全员 | Linux/macOS/Windows 验证 |
| 性能测试 | 后端 | 启动时间、资源占用 |
| 集成测试 | QA | 全流程测试 |
| 文档编写 | 全员 | 用户手册、开发文档 |

**里程碑**：Claw One MVP 发布

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **OpenClaw 版本不兼容** | 中 | 高 | 版本锁定、社区沟通 |
| **开发进度延迟** | 高 | 中 | 敏捷迭代、范围裁剪 |
| **用户反馈不及预期** | 中 | 高 | 早期用户参与、快速迭代 |
